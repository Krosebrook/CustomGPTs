# Contributing to CustomGPTs

Thank you for your interest in contributing to the CustomGPTs repository! This guide will help you submit high-quality prompts and custom GPT configurations.

## 📋 Contribution Guidelines

### Before Contributing
- Check if a similar prompt or GPT already exists
- Ensure your contribution adds unique value
- Test your prompt/GPT thoroughly before submitting

### What to Contribute
- **Prompts**: Well-crafted prompts for specific use cases
- **Custom GPTs**: Complete GPT configurations with clear documentation
- **Improvements**: Enhancements to existing content
- **Documentation**: Better explanations, examples, or organization

## 🚀 How to Contribute

### Step 1: Fork and Clone
1. Fork this repository to your GitHub account
2. Clone your fork locally
3. Create a new branch: `git checkout -b feature/your-contribution-name`

### Step 2: Choose Your Contribution Type

#### For Prompts
1. Navigate to the appropriate category in `/prompts/`
2. Copy the [prompt template](templates/prompt-template.md)
3. Create a new file: `your-prompt-name.md`
4. Fill out all sections of the template

#### For Custom GPTs
1. Navigate to the appropriate category in `/custom-gpts/`
2. Copy the [custom GPT template](templates/custom-gpt-template.md)
3. Create a new file: `your-gpt-name.md`
4. Include complete configuration details

### Step 3: Quality Standards

#### Prompt Requirements
- [ ] Clear, specific instructions
- [ ] At least 2 usage examples
- [ ] Appropriate category placement
- [ ] Tested and verified outputs
- [ ] Proper markdown formatting

#### Custom GPT Requirements
- [ ] Complete system prompt included
- [ ] All configuration settings specified
- [ ] Setup instructions provided
- [ ] Example conversations included
- [ ] Performance notes documented

### Step 4: File Naming
- Use lowercase with hyphens: `my-awesome-prompt.md`
- Be descriptive but concise
- Avoid special characters except hyphens

### Step 5: Submit Pull Request
1. Commit your changes: `git commit -m "Add [type]: [brief description]"`
2. Push to your fork: `git push origin feature/your-contribution-name`
3. Create a pull request with:
   - Clear title describing the contribution
   - Detailed description of what you're adding
   - Any testing you've performed

## 📂 Directory Structure

### Prompts Categories
- **`prompts/creative/`**: Writing, art, creative projects
- **`prompts/business/`**: Marketing, strategy, productivity
- **`prompts/educational/`**: Learning, teaching, explanations
- **`prompts/technical/`**: Programming, engineering, technical tasks
- **`prompts/general/`**: Everyday tasks, general assistance

### Custom GPT Categories
- **`custom-gpts/assistants/`**: General-purpose AI helpers
- **`custom-gpts/specialists/`**: Domain-specific experts (medical, legal, etc.)
- **`custom-gpts/tools/`**: Task-specific utilities (calculators, converters, etc.)

## ✅ Review Process

### What We Look For
- **Quality**: Well-written, clear, and useful
- **Originality**: Adds new value to the repository
- **Documentation**: Complete and follows templates
- **Testing**: Evidence of validation and testing

### Review Timeline
- Initial review: 1-3 business days
- Feedback incorporation: As needed
- Final approval: 1-2 business days after requirements are met

## 🎯 Best Practices

### Writing Effective Prompts
- Be specific and detailed
- Include context and constraints
- Provide clear examples
- Test with different inputs
- Consider edge cases

### Creating Custom GPTs
- Start with a clear purpose
- Write comprehensive system prompts
- Document all settings and configurations
- Include varied conversation examples
- Test thoroughly before submitting

### Documentation Standards
- Use clear, professional language
- Include all required template sections
- Provide practical examples
- Add helpful customization tips
- Tag appropriately for discoverability

## 🚫 What Not to Contribute

- Prompts that violate OpenAI's usage policies
- Copyrighted content without permission
- Offensive, harmful, or inappropriate content
- Incomplete or untested configurations
- Duplicate content without significant improvements

## 🤝 Community Standards

- Be respectful and constructive
- Provide helpful feedback on others' contributions
- Follow the code of conduct
- Help maintain repository organization
- Share knowledge and learn from others

## 📞 Getting Help

- Open an issue for questions or clarifications
- Review existing prompts and GPTs for inspiration
- Ask for feedback during the contribution process
- Join discussions in pull request comments
